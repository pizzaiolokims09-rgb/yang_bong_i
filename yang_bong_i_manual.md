# 🇰🇷 양봉이(Yangbong-i) 트레이딩 봇 통합 매뉴얼 (V1.2)

상사님(꾸울~벅지!), 요청하신 대로 줄이 삐뚤어지는 표 형식을 모두 제거하고, 한눈에 쏙 들어오는 리스트 형식으로 매뉴얼을 전면 개편했습니다. 파일명도 지시하신 대로 `yang_bong_i_manual.md`로 변경 완료했습니다!

---

## 1. 봇 개요 (Bot Overview)
양봉이는 **한국투자증권(KIS) API**를 기반으로 작동하는 **실전 투자형 자산 리밸런싱 봇**입니다. 단순히 가격이 오르면 팔고 내리면 사는 것을 넘어, 절세(TLH), 배당금 재투자(DRIP), 위기 방어(Brake) 등 전문적인 퀀트 전략을 수행합니다.

## 2. 핵심 아키텍처 및 모듈 구성

### 🧬 메인 엔진 (Main Loop) - `main.py`
- **역할**: 봇의 심장부로 모든 스케줄링과 비동기 루프를 관리합니다.
- **주요 기능**:
    - **15분 감시 루프**: 15분마다 시장 시세와 잔고를 체크합니다.
    - **수면 모드 (Hibernation)**: 밤 11시 30분~새벽 6시 사이 증권사 서버 점검 시 봇을 잠시 재워 장애를 방지합니다.
    - **안전 부팅 (Boot Sequence)**: 봇이 재시작될 때 미체결 주문을 싹 정리하고 Rate Limiter 데이터를 강탈(Sync)해 일관성을 유지합니다.

### ⚖️ 지능형 리밸런서 - `rebalancer.py`
- **역할**: 목표 비중과 현재 잔고를 비교하여 최적의 매매 시나리오를 설계합니다.
- **핵심 로직**:
    - **증분 리밸런싱**: 전체 매도 후 재매수가 아닌, **목표치와의 차액(Diff)**만큼만 매매하여 수수료를 절감합니다.
    - **절세 시퀀스 (TLH)**: 익절할 때 발생하는 세금을 막기 위해 손실 중인 종목을 먼저 매도/재매입하여 세금을 상쇄합니다.
    - **기타 자산 청소 (Junk Clear)**: 설정(`portfolio_config.json`)에 없는 종목은 자동으로 전량 매도하여 예수금을 확보합니다.
    - **예수금 감사관 (Sync Auditor)**: "매도해서 들어올 돈"까지 미리 계산하여 예수금 부족으로 매수가 멈추는 현상을 방지합니다.
    - **부분 매수 (Partial Buy)**: 가용 예수금이 전체 매수 필요 금액보다 부족할 경우, 전체를 차단하지 않고 금액이 작은 종목부터 우선 매수합니다. 예산 초과된 종목은 다음 리밸런싱 때 자동으로 재시도됩니다.
    - **더스트(Dust) 자산 스킵**: 목표 비중과의 차액이 1주 매수 금액보다 작을 경우, 잦은 매매로 인한 수수료 낭비를 막기 위해 해당 회차 매수를 스킵합니다.
    - **시드 자동 증액**: 현재 시드가 모든 종목을 최소 1주씩 매수하기에 부족하면, 계좌 내 미할당 여유 자산을 자동으로 시드에 편입합니다.
    - **대량 매매 분할 체결 (TWAP)**: 단일 종목 기준 500만 원 이상의 대규모 매수/매도 주문 시, 시장 충격(슬리피지)을 최소화하기 위해 10분 동안 5회에 걸쳐 자동으로 쪼개서 분할 주문을 넣습니다.
    - **시세 오류 방어**: 거래소 점검이나 API 지연으로 현재가가 0원으로 조회될 경우, 잘못된 비중 계산을 막기 위해 해당 종목을 리밸런싱에서 안전하게 제외합니다.

### 🚑 주문 앰뷸런스 & 구출 - `main.py`, `state_manager.py`
- **역할**: 시장가 주문 실패나 미체결 지정가를 관리합니다.
- **구출 로직**:
    - 시장가 주문 실패 시 즉시 **현재가 기준 지정가**로 전환하여 대기열에 등록합니다.
    - **15분 타임아웃**: 지정가가 15분 동안 체결되지 않으면 자동으로 취소하고 **시장가로 재투입**하여 반드시 체결시킵니다.

### 🛡️ 위기 방어 시스템 및 통신 복원력 (Crisis Management & Resilience)
- **변동성 브레이크**: 특정 종목이 자정 대비 **-7% 이상 폭락** 시, 추가 매수를 금지하고 보유량의 50%를 즉시 탈출(시장가 매도) 시킵니다. (단, VIX, 달러 인덱스 등 매크로 지표는 폭락 감지에서 제외하여 불필요한 알림을 방지합니다.)
- **Rate Limiter**: 하루 최대 주문 횟수(200회) 또는 시드의 400% 거래량 초과 시 봇을 즉시 정지(Panic Mode)시켜 무한 루프 사고를 방지합니다.
- **AI 통신 장애 대응 (Fallback)**: 구글 Gemini 서버의 지연(Timeout)이나 503 오류 발생 시, 10~15초 간격으로 최대 3회 자동 재시도하며, 마지막 시도에는 더 가볍고 빠른 Flash 모델로 우회 접속하여 시스템 다운을 막습니다. 통신이 완전히 끊기면 잘못된 리밸런싱 지시(HOLD 등) 대신 명확한 "ERROR(통신 장애)"를 텔레그램으로 보고합니다.

### 🦅 시장 구조 감시 시스템 (Market Sentinel) - `market_sentinel.py`
- **역할**: 장 중 15분 간격으로 한국 시장 특유의 수급 불균형과 프로그램 매도 폭탄을 감시하여 조기 경보를 울립니다.
- **핵심 기능**:
    - **수급 불균형 감시**: 외국인의 현물 매수(>1,000억)와 선물 매도(>5,000계약)가 동시에 일어나는 차익거래 스탠스 포착.
    - **포지션 역전 감시**: 외국인 현물 매수 금액보다 선물 매도(환산금액)가 커지는 양매도/헤지 구간 경고 (5분 이상 지속 시).
    - **프로그램 매도 폭탄 감지 (Bottom Fishing)**: KOSPI200 지수가 급락(-1% 이상)하고 비차익 순매도 비중이 60% 이상일 때 '기계적 투매'로 간주하여 자동 '저점 매수 탐색 모드' 발동.
    - **만기일 알림 스케줄러**: 매월 둘째 주 목요일(선물/옵션 만기일)을 자동 계산하여, 2일 전, 1일 전, 당일 아침에 변동성 확대 경고 알림 발송 (3/6/9/12월 쿼드러플 위칭데이 판별).
    - **가변 현금 비중 (Dynamic Exposure)**: 외국인의 누적 선물 순매도 수량(10,000계약 단위)에 비례하여 최대 목표 현물 비중 한도를 축소시켜 하락장에 선제적으로 대비.

### 🎯 스마트 배당금 투자 (Smart DRIP)
- **역할**: 입금된 배당금을 놀리지 않고 즉시 재투자합니다.
- **로직**: 현금이 1만원 이상 급증하면, AI가 현재 포트폴리오 중 **가장 저평가된 종목**을 골라 즉시 1주라도 더 삽니다.

## 3. 설정 및 데이터 관리 시스템

### 📄 config.py (시스템 환경설정)
- **역할**: 봇의 뼈대가 되는 하드웨어적 설정을 담당합니다.
- **관리 항목**: 증권사 API Key/Secret (AES-256 암호화 저장), 계좌번호, 티커 매핑 사전(ASSET_TICKERS) 등
- **보안**: `.secret/env.enc` 파일로 암호화 관리. 평문 `.env` 파일은 디스크에 남기지 않고 메모리에서만 파싱합니다.

### 📄 portfolio_config.json (투자 전략 설정)
- **역할**: 상사님이 정해주신 포트폴리오의 실질적인 가이드라인입니다.
- **관리 항목**: 목표 종목 리스트 딕셔너리(`"assets": {"종목명": 비중, ...}` 구조), 동적 리밸런싱 타겟 비중.

### 📄 state.json (봇의 기억 장치)
- **역할**: 봇이 꺼졌다 켜져도 이전 상태를 기억하게 해주는 단기 기억 저장소입니다.
- **관리 항목**: 설정 시드 금액, 당일 누적 거래량, 위기 모드 발동 티커, 앰뷸런스 대기열 등

### 📄 meeting_history.json (대화 지침 저장소)
- **역할**: 상사님과의 대화에서 나온 특별 지시나 성격 변경 등을 기억합니다.
- **관리 항목**: 상사님의 커스텀 수동 명령, 봇의 말투 및 성격 페르소나 지침

## 4. 텔레그램 명령체계 (User UI)
- `/start`: 메인 메뉴 버튼을 열어 수동 리밸런싱, 상태 확인, 매매 중단 등의 기능에 접근합니다.
    - 🚀 수동 종목 즉시실행: 강제 리밸런싱 엔진 가동 (매도 우선 → Sync Auditor → 부분 매수)
    - 📊 종목 상태 확인: 현재 포트폴리오 분석 및 비중 비교 리포트
    - 🛑 전체 매매 긴급 중단: 킬스위치 발동
- `/status`: 현재 계좌 전체 수익 현황, 종목별 상세 비중 및 손익 리포트 출력
- `/panic`, `/stop`: 모든 매매 즉시 중단 (킬스위치)
- `자연어 대화`: 채팅을 통해 비중을 변경하거나 봇의 성격을 변경 (Gemini AI 연동)

## 5. 잔고 조회 구조 (broker_kis.py)
- **페이징 연속조회**: 보유 종목이 많을 경우 KIS API가 여러 페이지로 나눠 반환합니다. `ctx_area_fk100/nk100` 키로 다음 페이지를 자동 조회하며, 종목코드(`pdno`) 기준으로 중복을 제거합니다.
- **현금(예수금) 계산**: KIS API의 `dnca_tot_amt` 필드를 직접 사용합니다. 이 필드가 0이면 `nass_amt - stocks_value`로 역산합니다.
- **총 자산 계산**: `nass_amt`(순자산) 필드를 우선 사용합니다.
- **주요 API 필드 참고**:
    - `dnca_tot_amt`: 예수금 총액 (주문 가능 현금)
    - `scts_evlu_amt`: 유가증권 평가금액 (주식만)
    - `nass_amt` / `tot_evlu_amt`: 순자산 (주식 + 현금)

## 6. 사후 관리 및 업데이트 가이드
1. **코드 수정 시**: 필히 `rebalancer.py`의 `Sync Auditor` 로직과 `broker_kis.py`의 페이징 중복 방지 로직을 해치지 않도록 주의합니다.
2. **티커 변경 시**: `config.py`의 매핑 사전과 `portfolio_config.json`을 동시에 업데이트해야 인식이 원활합니다. 종목명이 정확히 일치해야 합니다.
3. **손익 문제 발생 시**: `broker_kis.py`의 `output2` 파싱 필드를 증권사 공지사항과 대조합니다.
4. **리밸런싱 실패 시**: `journalctl -u yangbong.service`로 로그를 확인합니다. `[잔고]`, `[Auditor]`, `[주문 계산 완료]` 키워드로 검색하면 문제 구간을 빠르게 찾을 수 있습니다.
